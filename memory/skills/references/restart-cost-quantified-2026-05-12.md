# 重启成本量化 — 2026-05-12 实案

## 背景

2026-05-12 在2.5小时内对同一现货网格Bot进行了5次重启：
- 10:12 旧4币网格首次重启（Phase 1a过渡）
- 12:33 XRP单币网格启动
- 16:10-16:30 V4.0→V4.1迭代期间4次重启

## 成本拆解

### 直接成本（每重启）

| 项目 | 估算值 | 说明 |
|:----|:------:|:-----|
| INIT买入手续费 | $0.02 | 3层×~$45×0.04% |
| 幽灵卖出手续费 | $0.02 | 3层幽灵清仓×0.04% |
| 买卖滑点 | $0.05-0.15 | 市价单的买卖价差 |
| **单次重启总成本** | **~$0.10-0.20** | |

### 间接成本（多币种场景）

如果旧网格有ETH/SOL/DOGE等多币种持仓：

| 币种 | 仓位 | 成本价 | 幽灵清仓价 | 价差亏损 |

### 2026-05-12 总成本

```
直接手续费:     ~$0.40 (5次重启)
滑点损失:       ~$0.50 (5次)
幽灵清仓价差:   ~$6.29 (旧多币种持仓)

```

## 成本效益分析

```
重启次数  |  累计成本  |  策略价值
0         |  $0        |  V4.0参数（14层×$700）
5         |  ~$14      |  V4.1参数（20层×$1,000）
```

5次重启花费$14，只换来了层数增加6层（14→20）。如果一次性改完重启，成本是~$0.20而非$14。

## 铁则

1. **每次重启前估算成本**：手续费($0.04) + 幽灵清仓($0.10+)
2. **有旧浮仓时成本×10**：ETH/SOL/DOGE等会被幽灵清仓 → 价差亏损可能是手续费的50倍
3. **一天内重启>2次必须停止**：停下来重新设计方案，不要抱着"再试一次"的心态
4. **多币种→单币种的切换成本最高**：等价于清仓所有旧币种重新部署
