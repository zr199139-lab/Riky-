## 持仓快照

4个开放持仓（来自whale_hedge_positions.json）：
1. oc_1777861110: SELL follow @$0.54, hedge @$0.45, locked_pnl=$0.01
2. oc_1777871984: SELL follow @$0.6595, hedge @$0.2724, locked_pnl=-$0.0136
3. oc_1777874658: SELL follow @$0.662, hedge @$0.2704, locked_pnl=-$0.0135
4. oc_1777891742: SELL follow @$0.6595, CLOSED (超时) @$0.0000

## 根因

1. Bot跟单了鲸鱼在低流动性市场（预测市场）的挂单
2. 跟单后市场流动性不足，卖价(bid)固定为$0.3400
3. 跟单均价(fp)固定为$0.6595
4. 价差$0.0005（fp - bid = 0.6595-0.3400-0.3195... wait, 实际是profit calculation）
   - 实际上 fp=0.6595, bid=0.3400, total=0.9995, profit=0.0005=$0.0005
   - 这意味着: 买入价0.6595, 卖价0.3400（非常宽的价差，但方向不同）
   - 实际上是FP price和bid price的总和接近1.0，利润$0.0005
5. MIN_PROFIT=$0.01 但利润仅$0.0005，永远达不到阈值
6. 没有超时机制，Bot会永远循环检查

## 影响

- 约2小时无实际交易
- 资本$47.62全部锁定在4个持仓中
- PnL -$1.19（手续费+微亏）
- 大量API轮询消耗额度
- 最后1个持仓在19:06因"超时"平仓(PnL=$0.00)

## 发生的链式原因

1. Whale Hedge Bot在PAPER阶段跑通，但切换到LIVE后交易对手减少
2. 鲸鱼开仓后离场，留下Bot独自持有头寸
3. 预测市场某些类别（esports/weather/sports）流动性差
4. Bot的设计假设是价差会恢复，但实际不会

## 当前状态

- Bot仍在运行（PID存活，日志持续更新）
- 持仓逐步超时闭合，但过程极慢
- 需添加Spread Starvation检测 + 强制放弃机制
