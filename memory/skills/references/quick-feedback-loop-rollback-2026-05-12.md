## 部署新策略的"快速回滚"检查清单

在部署任何新策略之前，先想好：

- [ ] 如果1小时内被砍，资金退路在哪？
- [ ] 主网格是否在运行（保障不会所有Bot同时死）？
- [ ] 新策略的state文件是否独立于主网格？
- [ ] 新策略PID记录了吗？（方便快速kill）
- [ ] 回滚后是否需要重启主网格？

## 原则

1. 新策略=试错。试错成本必须控制在$1以内。
2. 用户说"不行"=立即停，不等。
3. 回滚到已知赚钱的策略=当天最安全的操作。
4. 不要解释为什么不行——直接执行回滚。
